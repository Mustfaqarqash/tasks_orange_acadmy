Inline vs Block Elements

An inline element occupies only the space bounded by the tags that define the inline element, whereas a block element occupies the entire space of its parent element, thus creating a "block" of content.

What we learn:

  1. The difference in space occupation between inline and block elements.
  2. How inline elements flow within text content.
  3. How block elements create new blocks of content.
  • Inline elements do not start on a new line.
  • Block elements start on a new line.
  • Inline elements do not force a new line to begin in the document flow.
  • Block elements force a new line to begin after the block.

HTML5 Semantic Elements

HTML5 introduced new semantic elements like <header>, <footer>, <section>, and <article> which provide a better structure and meaning to web documents, aiding accessibility and SEO.

What we learn:

  1. The purpose of semantic HTML elements.
  2. How semantic elements improve document structure and accessibility.
  3. How semantic elements contribute to better SEO.
  • Semantic elements provide meaning to the content.
  • They improve accessibility by providing context to assistive technologies.
  • They help search engines better understand the content and improve SEO.
